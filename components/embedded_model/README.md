# Embedded Model Component

This component contains the replaceable XGBoost model generated by `m2cgen`.

The preprocessing contract is fixed in `main/data_preprocessor.c`. As long as
the model keeps the same 38 features in the same order, only the files in this
component need to be replaced when testing new XGBoost hyperparameters.

## Replace The Model

From the `Embedded_AI` repository, generate the C model:

```bash
.venv/bin/python models/binary_classification/output/export_xgboost_m2cgen.py \
  --joblib models/binary_classification/output/xgboost_binario_error_detector.joblib
```

Then copy these generated files into this component:

```text
xgboost_model_m2cgen.c
xgboost_error_postprocess.c
xgboost_error_postprocess.h
xgboost_feature_order.h
```

Expected firmware layout:

```text
components/embedded_model/
  CMakeLists.txt
  README.md
  xgboost_model_m2cgen.c
  xgboost_error_postprocess.c
  include/
    xgboost_error_postprocess.h
    xgboost_feature_order.h
```

## Usage

After `data_preprocessor_get_center_feature_vector()` returns `ESP_OK`, the
firmware can call:

```c
double probability = xgboost_predict_error_probability(features);
bool has_error = xgboost_predict_has_error(features);
```

`xgboost_predict_has_error()` applies the threshold stored in
`xgboost_error_postprocess.h`.
